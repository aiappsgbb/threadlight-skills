One pattern · six sectors

Six industries. One pattern.

The same five-stage funnel ships agents across regulated and unregulated industries. The skills do not change — the SPEC does. Below: one representative pilot per sector we work in, with the skills it leans on and the talk track that opens the door.

Financial services

Claims triage — route by complexity & regulator exposure.

Inbound claims (insurance, retail banking disputes) are scored for complexity and regulator exposure, then routed to the right desk with a structured pre-fill. Spec, deploy, evaluate — in-session.

Pilot in one paragraph

The agent reads the inbound claim, extracts the structured fields the adjuster wants, scores complexity, and produces a first-pass routing recommendation with the rationale. HITL on high-exposure cases.

Skills it leans on

threadlight-design · threadlight-deploy · threadlight-safe-check · threadlight-hitl-patterns

What we measure

  • Triage time per claim
  • Routing accuracy vs the desk's own backbook
  • HITL gate hit-rate (target band, not zero)

Talk track

“Show me one week of inbound claims. We can have the spec drafted and an agent running on synthetic data in the same session.”

Healthcare

Prior-authorization assistant — structured payor submissions.

The agent reads the clinical narrative and generates the structured payor submission — with HITL gates wired before anything leaves the building.

Pilot in one paragraph

Clinician's free-text note in; structured payor submission out. The agent never sends; it stages, scores its own confidence, and surfaces gaps for the human reviewer.

Skills it leans on

threadlight-design · threadlight-hitl-patterns · threadlight-deploy · threadlight-production-ready

What we measure

  • Reviewer minutes saved per submission
  • Submission rejection rate vs baseline
  • Audit trail completeness (100% required)

Talk track

“The agent does the typing. The clinician keeps the judgement. Audit trail and HITL gates are wired before deploy — not after.”

Manufacturing

Field-service knowledge agent — grounded in the manuals.

Technicians ask the agent. The agent grounds against equipment manuals + service history — with cost-projected SKUs picked before go-live so the bill is the bill.

Pilot in one paragraph

The agent indexes equipment manuals + the customer's past service tickets. A technician on the floor asks a natural-language question and gets a grounded answer with the source paragraph inline.

Skills it leans on

threadlight-design · threadlight-deploy · threadlight-consumption-iq

What we measure

  • First-call-resolution rate
  • Time-to-answer on the floor
  • Projected monthly cost vs production load

Talk track

“Cost-projection runs before deploy. The customer sees the bill before they sign for it.”

Retail

Merchandising co-pilot — promo plans grounded in real lift.

The buyer asks for a promo plan; the agent drafts it grounded in last-quarter SKU performance and store-cluster lift. The buyer keeps the pen; the agent keeps the maths.

Pilot in one paragraph

The agent ingests SKU-level sales + store cluster metadata, then drafts promo plans that name the lift it expects and the confidence interval. The buyer accepts, edits, or rejects each line.

Skills it leans on

threadlight-design · threadlight-demo-data-factory · threadlight-deploy

What we measure

  • Buyer minutes per plan
  • Plan acceptance rate by category
  • Forecast lift vs realized lift (post-mortem)

Talk track

“The agent shows the maths. The buyer keeps the judgement. We run the agent against your own back-book before you commit a single SKU.”

Telco

Network-incident triage — first-pass remediation in minutes.

The agent cross-references SNMP, ticket history, and runbooks into a first-pass remediation — cost-projected against the production load profile so the NOC manager can size it before rollout.

Pilot in one paragraph

Pager fires. The agent assembles SNMP context, recent tickets on the same device, and the relevant runbook section into a single first-pass remediation. The NOC engineer approves, edits, or rejects.

Skills it leans on

threadlight-design · threadlight-deploy · threadlight-consumption-iq · threadlight-hitl-patterns

What we measure

  • Time from pager to first-pass remediation
  • Engineer acceptance rate per incident class
  • Projected cost at production incident volume

Talk track

“The agent doesn't decide. It assembles. The engineer keeps the keys to the network.”

Public sector

Citizen case-status agent — plain-language updates with audit.

Public-language status updates with case-worker hand-off built in. Governance and audit trail are non-negotiable; the chain refuses to ship without them.

Pilot in one paragraph

The agent reads the case-management system, summarises the current status in citizen-readable language, and offers an escalation path to a human case-worker. Every reply is logged.

Skills it leans on

threadlight-design · threadlight-hitl-patterns · threadlight-production-ready

What we measure

  • Reduction in inbound contact-centre volume
  • Escalation rate to human case-workers
  • Audit-trail completeness (100% required)

Talk track

“Audit and HITL come before deploy. The chain refuses to ship a public-facing agent without them — that's a feature, not a friction.”

Same chain. Different SPEC.

The six pilots above all walk the same five-stage funnel. What changes is the SPEC — the customer's process, their data, their regulator. The skills do not change.
