The skills chain

You prompt. The coding agent picks up the right skills.

Skills are composable building blocks — Markdown instructions that any coding agent (GitHub Copilot CLI, VS Code, Claude Code) reads and follows. You describe what you need in a prompt. The agent discovers the matching skill, runs it, and hands you the artefacts — all the way to a governed agent on your Azure with a paved path to production via AI Citadel and AGT.

What it looks like in the field

Six industries. One pattern.

The same five-stage funnel ships agents across regulated and unregulated industries. The skills don't change — the SPEC does. Below: one example process per industry we work in.

🏦

Financial services

Claims triage

Route inbound claims by complexity and regulator exposure. Spec, deploy, evaluate — in-session.

spec → deploy → safe-check 🏥

Healthcare

Prior-authorization assistant

Generates structured payor submissions from the clinical narrative — with HITL gates wired from day zero.

spec → HITL patterns → deploy 🏭

Manufacturing

Field-service knowledge agent

Technicians ask the agent. The agent grounds against equipment manuals + service history — with cost-projected SKUs picked before go-live.

spec → deploy → cost 🛍️

Retail

Merchandising co-pilot

Buyer asks for promo plans; agent drafts them grounded in last-quarter SKU performance and store-cluster lift.

spec → demo-data → deploy 🏛️

Public sector

Citizen-facing case status

Public-language status updates with case-worker hand-off built in. Governance and audit trail are non-negotiable; the chain refuses to ship without them.

spec → HITL → production-ready 📡

Telco

Network-incident triage

Cross-references SNMP, ticket history, and runbooks into a first-pass remediation — cost-projected against the production load profile.

spec → deploy → consumption-iq

Demo first, build second

Start in Kratos. Build in Threadlight.

Two stages of the same journey, not two competing products. Kratos is the no-code quickstart — pick a persona, wire tools, talk to a working agent in the browser, all in the first ten minutes of a customer conversation. Threadlight is the pro-code path that follows — when the demo turns into a commitment, the skills on this page take that idea from one paragraph to a Foundry-hosted agent with evals, observability, cost projection, and a customer hand-off pack.

  • Kratos — first ten minutes, zero code. Try a working agent live, customize persona and tools in-app, export the spec when the conversation gets serious.
  • Threadlight — what follows, when it gets real. Take that spec (or a fresh prompt) all the way through specify → design → build → safe-check → deploy — opinionated, pro-code, production-grade.
  • Same Foundry. Same governance. What you demo in Kratos is what you build with Threadlight — one hosted-agent runtime, one set of hooks for hand-off, telemetry, and cost.
