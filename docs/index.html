Paved path to production

Green safe-check isn’t go-live.

A pilot that passes threadlight-safe-check --phase post-deploy is structurally complete — every selector landed, every channel reaches, no placeholder image. That is not the same as ready for the customer architecture review. Without an evidence-backed answer to "what posture is this in, what’s missing, who owns each gap, can we go live with waivers?" — pilots quietly settle into the lab graveyard.

threadlight-production-ready produces that answer in one command — across three postures and thirteen cross-cutting pillars, soft-advisory (never fails a build), gracefully degrading to not-verified when Azure permissions are missing.

Engineering invariants

Five guardrails the chain refuses to ship without.

These aren’t opinions. They are gates encoded in the skills. If any one of them is missing, the gate fails and the pilot doesn’t leave the lab.

🔑
id

Keyless identity, end-to-end.

DefaultAzureCredential everywhere. UAMI for the agent, the MCP server, the workspace, Cosmos, AI Search. Zero API keys committed to source. Zero secrets in environment variables.

Owned by foundry-hosted-agents · threadlight-deploy
📊
obs

Telemetry from minute one.

App Insights + Log Analytics in Bicep. Foundry account-level connection so hosted agents auto-inject the connection string. configure_azure_monitor() wrapped for local-dev safety. The first trace lands before the first customer call.

Owned by foundry-observability
📎
ci

Citation-grounded answers.

Every answer carries at least one citation. Every claim about an account, a policy, a comms-template version is reproducible. Refusals are explicit (“no matching content”) rather than confabulated.

Owned by per-process AGENTS.md · enforced by foundry-evals
📜
au

Audit-grade trail.

Append-only audit emit on every invocation. Cosmos hot tier for 90 days; cool tier 90 d→1 y; ADLS archive 1 y→7 y. The FCA examiner question pattern is answerable on day one, not day ninety.

Owned by SPEC § 4.7 · emitted via the agent runtime
✅
ev

Evals from day one.

SPEC § 9 eval scenarios are generated from BR-XXX rules. The eval dataset shape carries tool_calls + tool_outputs so the scorer can see what the agent read. Continuous-eval baseline ships with the deploy.

Owned by foundry-evals
