An AI assistant that triages retail returns in under a minute. Given a return (RMA)
or order id, it correlates the order, the return record, and the customer profile,
applies Contoso Retail's return policy, and recommends one of four outcomes —
**approve refund**, **deny refund**, **escalate to supervisor**, or **request more
info** — each with a cited policy clause and an audit record.

> Generated by the `threadlight-design` skill in **Fast-PoC mode**. This is the
> from-scratch design layer: `specs/`, `AGENTS.md`, and `src/agent/skills/`. Deploy
> artifacts are produced separately by `threadlight-deploy`.

## What it does (spec § 1)

- **Actors**: customer-service agent (runs it), returns supervisor (handles
  escalations), the AI assistant.
- **Upstream systems** (all mocked for the PoC): order-management system,
  returns-database, customer-profile service.
- **Decisions**: `approve_refund` · `deny_refund` · `escalate_to_supervisor` ·
  `request_more_info`.

## Architecture (text)

```
CS agent / Workspace / Teams
        │  (RMA or order id)
        ▼
  Returns Triage assistant (gpt-5.4, via Citadel APIM gateway)
        │
        ├─ intake-validation ──► oms_get_order · returns_get_case · customer_get_profile
        ├─ policy-eligibility ─► Foundry IQ (return policy, cited)
        ├─ fraud-escalation ───► value ceiling + serial-returner gate
        └─ disposition-decision ► returns_apply_decision (writes decision + audit)
                                        │
                          escalation ──►  Returns supervisor (Teams card)
```
