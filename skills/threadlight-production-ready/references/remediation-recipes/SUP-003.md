## Target file
One of:
- `package-lock.json` (Node.js / npm)
- `yarn.lock` (Yarn)
- `poetry.lock` (Python / Poetry)
- `Pipfile.lock` (Python / Pipenv)
- `requirements.txt` (Python / pip, with pinned versions)
- `.csproj` (C# / .NET, version pins in project file)

Choose the tool that matches your dependency-management approach.

## Edit type
`insert` (generate + commit)

## Edit recipe

**For Node.js / npm:**
1. Ensure `package.json` exists with your dependencies listed.
2. Generate lock file:
   ```bash
   npm install --save-exact
   ```
3. Verify `package-lock.json` is created and all versions are pinned (e.g., `"version": "1.2.3"`, not `"^1.2.3"`).
4. Commit:
   ```bash
   git add package-lock.json
   git commit -m "chore: commit dependency lock file (package-lock.json)"
   ```

**For Yarn:**
1. Ensure `package.json` exists.
2. Generate lock file:
   ```bash
   yarn install --frozen-lockfile
   ```
3. Verify `yarn.lock` is created.
4. Commit:
   ```bash
   git add yarn.lock
   git commit -m "chore: commit dependency lock file (yarn.lock)"
   ```

**For Python / Poetry:**
1. Ensure `pyproject.toml` exists with dependencies listed.
2. Generate lock file:
   ```bash
   poetry lock --no-update
   ```
3. Verify `poetry.lock` is created with pinned versions.
4. Commit:
   ```bash
   git add poetry.lock
   git commit -m "chore: commit dependency lock file (poetry.lock)"
   ```

**For Python / Pipenv:**
1. Ensure `Pipfile` exists.
2. Generate lock file:
   ```bash
   pipenv lock
   ```
3. Verify `Pipfile.lock` is created.
4. Commit:
   ```bash
   git add Pipfile.lock
   git commit -m "chore: commit dependency lock file (Pipfile.lock)"
   ```

**For Python / pip (no lock file generator):**
1. Generate a pinned `requirements.txt` from the current environment:
   ```bash
   pip freeze > requirements.txt
   ```
2. Review `requirements.txt` and remove any packages not in your actual dependency list.
3. Ensure all versions are pinned exactly (e.g., `flask==2.3.2`, not `flask>=2.3`).
4. Commit:
   ```bash
   git add requirements.txt
   git commit -m "chore: commit pinned dependency manifest (requirements.txt)"
   ```

**For C# / .NET:**
1. Edit `.csproj` to pin package versions in `<PackageReference>` elements:
   ```xml
   <ItemGroup>
     <PackageReference Include="Newtonsoft.Json" Version="13.0.3" />
   </ItemGroup>
   ```
2. Ensure no `*` or version ranges (e.g., `[1.0, 2.0)`) exist — use exact versions.
3. Commit the `.csproj` file.
