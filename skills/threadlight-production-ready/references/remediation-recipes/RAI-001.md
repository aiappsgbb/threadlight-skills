## Target file
`infra/foundry/deployments.bicep` (or the canonical Foundry model deployment file — search for `Microsoft.CognitiveServices/accounts/deployments` if the path differs).

## Edit type
`replace`

## Edit recipe
1. Locate the `Microsoft.CognitiveServices/accounts/deployments` resource (typically named `foundry-gpt4`, `model-deployment`, or similar).
2. Inside the `properties` block, find or add the `raiPolicyName` property. If it does not exist, add it:

   ```bicep
   properties: {
     model: {
       format: 'OpenAI'
       name: 'gpt-4'
       version: '1106'
     }
     scaleSettings: {
       scaleType: 'Standard'
     }
     raiPolicyName: 'medium-content-filter'
   }
   ```

3. If `raiPolicyName` already exists but is empty or set to an empty string, replace it with a valid filter name: `'low-content-filter'`, `'medium-content-filter'`, or `'high-content-filter'` (depending on security posture).
4. Ensure the content filter resource (`Microsoft.CognitiveServices/accounts/raiPolicies`) is defined in the same Bicep file or referenced from a shared module. RAI-101 will validate the resource is live.

## Verification
1. Deploy the updated Bicep to the target resource group:
   ```bash
   az deployment group create \
     --resource-group <RG> \
     --template-file infra/foundry/deployments.bicep \
     --parameters <params.json or inline params>
   ```

2. Re-run threadlight: `python3 skills/threadlight-production-ready/scripts/production_ready.py --target-rg <RG> --target-sub <SUB>`. RAI-001 should flip from `fail` to `pass` when `raiPolicyName` is detected in the Bicep source.
