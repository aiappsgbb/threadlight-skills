target_file: Dockerfile

## Target file
`Dockerfile` (or `docker-compose.yml`, `infra/aca.bicep`, `infra/main.bicep` — search for any image references ending in `:latest`, `:tag` without digest)

## Edit type
`replace`

## Edit recipe
1. Search all Dockerfile(s) and infrastructure-as-code files for image references:
   ```bash
   grep -r "FROM\|image:" --include="Dockerfile" --include="*.bicep" --include="docker-compose.yml"
   ```

2. For each image reference, replace the tag with a digest pin:
   - **Before:** `FROM python:3.11` or `image: myregistry.azurecr.io/app:v1.0`
   - **After:** `FROM python:3.11@sha256:abc123...` or `image: myregistry.azurecr.io/app:v1.0@sha256:abc123...`

3. To obtain the digest for an image:
   ```bash
   # For public images (Docker Hub)
   docker pull python:3.11
   docker inspect --format='{{.RepoDigests}}' python:3.11

   # For ACR images
   az acr repository show --name <acr-name> --image <image>:<tag> --query "digest"
   ```

4. Example Dockerfile before and after:

**Before:**
```dockerfile
FROM python:3.11
FROM node:18-alpine
COPY . /app
```

**After:**
```dockerfile
FROM python:3.11@sha256:a1b2c3d4e5f6...
FROM node:18-alpine@sha256:x9y8z7w6v5u4...
COPY . /app
```

5. For Bicep (infra/aca.bicep or similar):

**Before:**
```bicep
properties: {
  containers: [
    {
      image: 'myregistry.azurecr.io/myapp:latest'
    }
  ]
}
```

**After:**
```bicep
properties: {
  containers: [
    {
      image: 'myregistry.azurecr.io/myapp@sha256:a1b2c3d4e5f6...'
    }
  ]
}
```

6. Remove any references to `:latest` entirely — digest pins do not support floating tags.

## Verification
Re-run threadlight: `python3 scripts/production_ready.py --target-rg <RG> --target-sub <SUB>`. SUP-001 should flip from `fail` to `pass`.

Manual check:
```bash
grep -r "@sha256" Dockerfile infra/*.bicep | wc -l
```

Expected: All image references include a `@sha256:` digest. No `:latest` or undigested tags remain.
