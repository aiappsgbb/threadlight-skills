## Target file
`infra/observability/action-group.bicep` (or wherever `Microsoft.Insights/actionGroups` is declared — search the repo if this path differs)

## Edit type
`replace`

## Edit recipe
1. Locate the `Microsoft.Insights/actionGroups` resource in your Bicep file.
2. Inside `properties.groupShortName` and `properties.receivers`, ensure at least one **non-noreply** destination is present:
   - **Teams webhook:** `teamWebhook` receiver targeting the production alerts channel
   - **PagerDuty:** `automationRunbook` or third-party webhook integration
   - **Opsgenie:** HTTPS webhook to on-call rotation
   - **Real email group:** `emailReceiver` with a monitored shared inbox (not noreply@company.com)

Example Bicep configuration:

```bicep
resource actionGroup 'Microsoft.Insights/actionGroups@2023-01-01' = {
  name: 'ag-prod-alerts'
  location: 'global'
  properties: {
    groupShortName: 'prod-ag'
    receivers: [
      {
        name: 'teams-prod'
        teamWebhookReceivers: [
          {
            name: 'prod-incidents-channel'
            objectId: 'teams-channel-id'
            webhookUrl: 'https://outlook.webhook.office.com/webhookb2/...'
            useCommonAlertSchema: true
          }
        ]
      }
      {
        name: 'pagerduty'
        webhookReceivers: [
          {
            name: 'PagerDuty Integration'
            objectId: 'pdgty-receiver'
            serviceUri: 'https://events.pagerduty.com/v2/enqueue'
            useCommonAlertSchema: true
          }
        ]
      }
    ]
  }
}
```

3. **Verify the receiver:**
   - Teams: Confirm the webhook URL points to an active team / channel and `useCommonAlertSchema: true` is set.
   - PagerDuty / Opsgenie: Verify the integration key / webhook is in Key Vault and not expired.
   - Email: Use a monitored shared inbox, not a noreply address.

4. Remove any receivers that are `noreply@...` or disabled.

## Verification
Deploy the Bicep changes to your target resource group. Re-run threadlight: `python3 scripts/production_ready.py --target-rg <RG> --target-sub <SUB>`. SRE-101 should flip from `fail` to `pass`.

To manually verify:
```bash

```

Expected: At least one action group with a real webhook or email receiver (not noreply).
