## Target file
`infra/foundry/deployments.bicep` (or wherever the `Microsoft.CognitiveServices/accounts/deployments` resources are declared — search the repo for `@2024` and `deployments` if the canonical path differs).

## Edit type
`replace`

## Edit recipe
1. Locate all `Microsoft.CognitiveServices/accounts/deployments@<version>` resources in the Bicep.
2. For each deployment, find the `properties.model.version` field.
3. If `properties.model.version: 'latest'`, replace it with a specific date-string pinned version:

   ```bicep
   properties: {
     model: {
       format: 'OpenAI'
       name: 'gpt-4o'
       version: 'latest'  // ❌ CHANGE THIS
     }
   }
   ```

   to:

   ```bicep
   properties: {
     model: {
       format: 'OpenAI'
       name: 'gpt-4o'
       version: '2024-08-06'  // ✅ Use a pinned date-string
     }
   }
   ```

4. Use the date-string that corresponds to the model you intend to use (e.g. `2024-08-06`, `2024-11-20`). Consult Microsoft's Foundry model documentation or `az cognitiveservices account deployment list` to find available pinned versions.
5. If no pinned version is known, use the latest release date available in your region or the model's general availability date.
