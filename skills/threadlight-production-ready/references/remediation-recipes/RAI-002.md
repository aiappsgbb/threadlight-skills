## Target file
`policy.yaml` (the agent governance policy artefact — usually in the root or `infra/`).

## Edit type
`insert`

## Edit recipe
1. Locate the `policy.yaml` file. If it does not exist, create it at the repo root.
2. Inside the policy YAML structure, locate or create a top-level `responsible_ai` or `rai_policy` block. If the policy does not yet have this section, add it at the appropriate level (typically after `governance` or `security` blocks):

   ```yaml
   responsible_ai:
     content_safety:
       enabled: true
       severity_thresholds:
         hate: medium
         sexual: medium
         selfHarm: medium
         violence: medium
     jailbreak_protection: enabled
     prompt_shields:
       jailbreak: enabled
       indirect_attack: enabled
     pii_redaction: enabled
     redaction_scope:
       - email
       - phone
       - ssn
   ```

3. If a `rai_policy` or `content_safety` section already exists but is incomplete, fill in missing fields (especially `prompt_shields`, which is required for RAI-003).
4. Ensure the policy is committed to the repository and discoverable by the assessor.
