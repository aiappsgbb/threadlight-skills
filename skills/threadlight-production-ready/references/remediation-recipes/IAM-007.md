## Target file
`infra/identity.bicep` (or the resource that declares the agent identity), or the
optional `agent-identity.governance.json` manifest at the repo root.

## Edit type
`replace`

## Edit recipe
A non-human identity with no accountable human owner is orphaned the day its
author changes teams — nobody knows whether it can be retired, re-scoped, or
rotated. Attach an owner so every agent identity is answerable to a person.

1. **Tag the identity resource** with an owner (any of `owner`, `ownerEmail`, or
   `ManagedBy` satisfies the check):

   ```bicep
   resource agentId 'Microsoft.ManagedIdentity/userAssignedIdentities@2023-01-31' = {
     name: 'id-${agentName}'
     location: location
     tags: {
       owner: 'ai-platform@contoso.com'   // ADD — a real, monitored mailbox
       reviewBy: '2026-12-31'
     }
   }
   ```

2. **Or record ownership in the governance manifest** if you prefer to keep it out
   of infra — `agent-identity.governance.json` at the repo root:

   ```json
   {
     "subjects": [
       { "id": "id-agent", "owner": "ai-platform@contoso.com", "reviewBy": "2026-12-31" }
     ]
   }
   ```

3. Prefer a **team distribution list** over an individual, so ownership survives
   staff changes. If you register the identity through **entra-agent-id**, set the
   owner there and let this repo inherit it via the manifest.
