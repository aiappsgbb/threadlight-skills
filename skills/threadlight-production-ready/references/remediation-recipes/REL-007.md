## Target file
`docs/restore-drills/<YYYY-MM-DD>.md` (or `tests/restore-drill-<YYYY-MM-DD>.md` — any timestamped restore drill artefact discoverable in `docs/` or `tests/` with an ISO-8601 date suffix and `.md` extension).

## Edit type
`manual` — DO NOT auto-apply. A restore drill requires live execution in the target environment and manual evidence collection.

## Edit recipe
Present this to the operator before proceeding:

> **Restore Drill — Required Evidence**
>
> REL-007 mandates a restore drill artefact dated within the last 90 days. This cannot be automated; the operator must:
>
> 1. **Execute Restore**: Follow the backup-restore runbook (`docs/runbooks/backup-restore.md`) to restore from the latest backup to a test environment or secondary instance.
> 2. **Validate Data**: Confirm data integrity and application function on the restored deployment.
> 3. **Document Results**: Create a file at `docs/restore-drills/<YYYY-MM-DD>.md` (e.g., `docs/restore-drills/2024-01-15.md`) with:
>
>    ```markdown
>    # Restore Drill — <Date>
>    
>    **Executed by**: <operator name>
>    **Date**: <YYYY-MM-DD>
>    **Duration**: <e.g., "45 minutes">
>    **RTO Validation**: <actual restore time vs. declared RTO>
>    
>    ## Procedure
>    - Initiated restore from backup timestamp: <timestamp>
>    - Target environment: <e.g., "staging">
>    - Restore completed at: <timestamp>
>    
>    ## Validation
>    - Data row count: <before vs. after>
>    - Application startup: <success|failure>
>    - Critical transactions verified: <yes|no>
>    
>    ## Findings
>    <any issues encountered, gaps in runbook, etc.>
>    ```
>
> 4. **Commit & Tag**: Once the drill is documented, commit the file to the repository:
>    ```bash
>    git add docs/restore-drills/<YYYY-MM-DD>.md
>    git commit -m "docs: restore drill executed on <YYYY-MM-DD>"
>    git push
>    ```
>
> If the operator declines, mark this as `accepted-risk` in `waivers.yaml` with signed justification.

The agent awaits operator confirmation before proceeding. If the operator confirms, guide them through the drill and commit the artefact. If the operator declines, file a waiver.
