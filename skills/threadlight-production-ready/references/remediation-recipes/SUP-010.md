## Target file
Every MCP server config in the repo: `.mcp.json`, `mcp.json`, `.vscode/mcp.json`, `.cursor/mcp.json`, or the `mcpServers` block of any agent config the scan reported as an offender.

## Edit type
`replace`

## Edit recipe
A floating reference (`@latest`, a bare package name, or a `:tag` without a digest) lets the resolved server change under you between runs. Pin each offender to an immutable coordinate:

1. **npx / uvx packages** — pin to an exact version:
   - **Before:** `"args": ["-y", "@modelcontextprotocol/server-filesystem"]`
   - **After:** `"args": ["-y", "@modelcontextprotocol/server-filesystem@1.0.3"]`

2. **Docker images** — pin to an image digest, not a tag. Prefer an image you have promoted into your own **Azure Container Registry (ACR)** so provenance and retention are under your control:
   - **Before:** `"args": ["run", "-i", "mcr.microsoft.com/mcp/example:latest"]`
   - **After:** `"args": ["run", "-i", "myregistry.azurecr.io/mcp/example@sha256:<digest>"]`
   - Obtain the digest:
     ```bash
     az acr repository show \
       --name <acr-name> --image mcp/example:1.0.0 \
       --query "digest" -o tsv
     ```

3. **pip-installed servers** — pin with `==`:
   - **Before:** `"args": ["-m", "some_mcp_server"]` with `pip install some_mcp_server`
   - **After:** `pip install some_mcp_server==0.4.1`

4. Re-run the producer to confirm each server now reports `pinned: true`:
   ```bash
   python3 scripts/mcp_sbom.py --root . --out mcp-sbom.json
   ```

If you consume MCP servers through **foundry-toolbox**, pin at the toolbox manifest layer the same way — an exact version or an ACR digest — so the platform resolves the identical server on every deployment.
