## Target file
`infra/foundry/account.bicep` (canonical for Foundry/MachineLearningServices; search for `Microsoft.CognitiveServices/accounts` or `Microsoft.MachineLearningServices/workspaces` if the path differs. Alternatives: `infra/cognitive-services.bicep`, `infra/openai.bicep`, `infra/search.bicep`).

## Edit type
`replace`

## Edit recipe

1. Locate all AI service resources in the Bicep files. These include:
   - `Microsoft.CognitiveServices/accounts` (Foundry, Cognitive Services)
   - `Microsoft.MachineLearningServices/workspaces` (Foundry)
   - `Microsoft.Search/searchServices` (AI Search)
   - `Microsoft.OpenAI/accounts` (OpenAI)

2. For each AI resource, find or add the `properties` block and ensure it includes:

```bicep
properties: {
  publicNetworkAccess: 'Disabled'
  // ... other properties
}
```

3. If `publicNetworkAccess` is missing, add it. If it is set to `'Enabled'` or any other value, replace it with `'Disabled'`.
4. Example replacement for Foundry account:

```bicep
resource foundry 'Microsoft.CognitiveServices/accounts@2024-06-01-preview' = {
  name: foundryName
  location: location
  kind: 'OpenAI'
  sku: {
    name: foundrySkuName
  }
  properties: {
    publicNetworkAccess: 'Disabled'
    customSubdomainName: customSubdomainName
    // ... additional properties
  }
}
```

5. Repeat for all AI service resources in the Bicep files.

## Verification

Re-run threadlight: `python3 scripts/production_ready.py --target-rg <RG> --target-sub <SUB>`. NET-003 should flip from `must-fix` to `pass` once all AI service resources have `publicNetworkAccess: 'Disabled'` in the compiled ARM output.

If still failing after the edit:
- Verify all AI resource types include the `publicNetworkAccess` property.
- Check that the property value is exactly `'Disabled'` (case-sensitive).
- Run `az bicep build -f infra/main.bicep` and inspect the output to confirm the property is set correctly in the compiled ARM template.
