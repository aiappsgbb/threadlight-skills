## Target file
`infra/foundry/foundry.bicep` (or wherever the `Microsoft.CognitiveServices/accounts` resource is declared for the Foundry account; search for `appInsightsResourceId` in the Foundry account properties).

## Edit type
`insert`

## Edit recipe
1. Locate the Foundry account resource in the Bicep template (search for `Microsoft.CognitiveServices/accounts` or `kind: 'OpenAI'`).
2. In the account's `properties`, ensure `appInsightsResourceId` is set:

```bicep
properties: {
  // ... other properties ...
  appInsightsResourceId: appInsightsResourceId  // from OBS-001 output
}
```

3. If `appInsightsResourceId` is missing, add it to the properties block. The value should come from the App Insights resource created in OBS-001.
4. Pass the App Insights resource ID as a parameter or import it from the observability module:

```bicep
param appInsightsResourceId string = ''  // Add if not present
```

5. Ensure OBS-001 (appinsights.bicep) is linked in the main template and its output is passed to the Foundry account resource.

## Verification
Re-run threadlight: `python3 scripts/production_ready.py --target-rg <RG> --target-sub <SUB>`. OBS-101 should flip from `fail` to `pass`.

Confirm the connection is live at the Foundry account:
```bash
az cognitiveservices account show \
  --name <foundry-account-name> \
  --resource-group <RG> \
  --query 'properties.appInsightsResourceId'
```

Expected: non-empty, points to the App Insights resource ID from OBS-001.
