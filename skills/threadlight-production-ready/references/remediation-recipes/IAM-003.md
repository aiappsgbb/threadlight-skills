## Target file
`infra/identity.bicep` (or wherever `Microsoft.Authorization/roleAssignments` are declared). The check flags `scope: subscription()` as must-fix.

## Edit type
`replace`

## Edit recipe
1. Locate subscription-scoped role assignments:

   ```bicep
   // BEFORE (BAD):
   resource subScopeRoleAssign 'Microsoft.Authorization/roleAssignments@2022-04-01' = {
     name: guid(subscription().id, 'Contributor')
     scope: subscription()  // TOO BROAD
     properties: {
       principalId: workloadIdentityPrincipalId
       roleDefinitionId: '/subscriptions/${subscription().subscriptionId}/providers/Microsoft.Authorization/roleDefinitions/b24988ac-6180-42a0-ab88-20f7382dd24c'
     }
   }
   ```

2. Replace with resource-scoped assignments (narrow to RG or specific resource):

   ```bicep
   // AFTER (GOOD):
   // At resource group scope:
   resource rgScopeRoleAssign 'Microsoft.Authorization/roleAssignments@2022-04-01' = {
     name: guid(resourceGroup().id, 'Foundry-Agent', workloadIdentityPrincipalId)
     scope: resourceGroup()  // Narrow to RG
     properties: {
       principalId: workloadIdentityPrincipalId
       roleDefinitionId: '/subscriptions/${subscription().subscriptionId}/providers/Microsoft.Authorization/roleDefinitions/5c607ea1-c196-4661-b85d-dffdf0a93f3b'  // Foundry-Agent role
     }
   }

   // Or at specific resource scope (e.g., Cosmos DB):
   resource cosmosRoleAssign 'Microsoft.Authorization/roleAssignments@2022-04-01' = {
     name: guid(cosmosDb.id, 'Cosmos Data Contributor', workloadIdentityPrincipalId)
     scope: cosmosDb  // Specific resource
     properties: {
       principalId: workloadIdentityPrincipalId
       roleDefinitionId: '/subscriptions/${subscription().subscriptionId}/providers/Microsoft.Authorization/roleDefinitions/00db8cb6-8adf-4b41-b3fb-b0779a1138c5'  // Cosmos Data Contributor
     }
   }
   ```

3. Use least-privilege roles (e.g., `KV Secrets User`, `Storage Blob Data Reader`, `Cosmos Data Contributor`) instead of broad roles like `Contributor`.

4. If multiple resources, create separate role assignments for each with its own scope.
