## Target file
`src/app.py` (or any file containing `clientSecret`, `ConnectionStrings`, or `password=` patterns). The check regex scans all tracked repo files.

## Edit type
`replace`

## Edit recipe
1. Find and remove all occurrences of client secrets, e.g.:

   ```python
   # BEFORE (BAD):
   credential = ClientSecretCredential(
       tenant_id=os.getenv("AZURE_TENANT_ID"),
       client_id=os.getenv("AZURE_CLIENT_ID"),
       client_secret=os.getenv("AZURE_CLIENT_SECRET")  # REMOVE THIS
   )

   # AFTER (GOOD):
   credential = DefaultAzureCredential()  # Uses managed identity
   ```

2. If using connection strings with embedded secrets:

   ```python
   # BEFORE:
   conn = f"DefaultEndpointsProtocol=https;AccountName=...;AccountKey={os.getenv('STORAGE_KEY')}"

   # AFTER:
   conn = f"DefaultEndpointsProtocol=https;AccountName=...;SharedAccessSignature={credential.token}"
   # Or use managed identity directly without connection string
   ```

3. Update `infra/identity.bicep` to declare a user-assigned managed identity (reference IAM-002) and grant it the necessary roles.

4. Update `src/appsettings.json` to remove secret references and use Key Vault instead:

   ```json
   {
     "AzureAd": {
       "TenantId": "<tenant>",
       "ClientId": "<client-id>"
       // No ClientSecret here
     }
   }
   ```

5. **Outside the repo**, rotate the compromised secret in Azure:

   ```bash
   az ad sp credential reset --id <service-principal-id> --display-name "v0.4.0-rotation" --years 1
   ```

6. Verify no remaining secrets:

   ```bash
   git grep -i "clientsecret\|password=" --exclude-dir=.git | grep -v "Binary" | grep -v ".md"
   ```

## Verification
Re-run threadlight: `python3 scripts/production_ready.py --target-rg <RG> --target-sub <SUB>`. IAM-001 should flip from `fail` to `pass` (regex finds no `clientSecret`, `password=`, or `BEGIN PRIVATE KEY`).

Also confirm the managed identity approach works by testing the app locally:

```bash
az login
azd debug
```

The app should authenticate without explicit credential setup.
