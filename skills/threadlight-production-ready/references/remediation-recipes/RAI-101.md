## Target file
`infra/foundry/rai-policy.bicep` (create if it does not exist; alternatively, the RAI content filter resource may be inserted into an existing foundry-related Bicep file).

## Edit type
`insert`

## Edit recipe
1. Create or locate `infra/foundry/rai-policy.bicep`. If it does not exist, create the file with the following structure:

   ```bicep
   param location string = resourceGroup().location
   param cognitiveServicesAccountId string
   
   resource raiPolicy 'Microsoft.CognitiveServices/accounts/raiPolicies@2024-04-01-preview' = {
     name: '${last(split(cognitiveServicesAccountId, '/'))}/medium-content-filter'
     location: location
     properties: {
       basePolicyName: 'Azure OpenAI [medium-4]'
       mode: 'Async'
       contentFilters: [
         {
           name: 'hate'
           allowedContentLevel: 'medium'
           blocking: true
           enabled: true
         }
         {
           name: 'sexual'
           allowedContentLevel: 'medium'
           blocking: true
           enabled: true
         }
         {
           name: 'selfHarm'
           allowedContentLevel: 'medium'
           blocking: true
           enabled: true
         }
         {
           name: 'violence'
           allowedContentLevel: 'medium'
           blocking: true
           enabled: true
         }
       ]
       userBlockListName: null
     }
   }
   
   output raiPolicyId string = raiPolicy.id
   output raiPolicyName string = raiPolicy.name
   ```

2. Link this module from the main foundry deployment Bicep (e.g., `infra/main.bicep` or `infra/foundry/deployments.bicep`):

   ```bicep
   module raiPolicies 'infra/foundry/rai-policy.bicep' = {
     name: 'rai-policy-deployment'
     params: {
       location: location
       cognitiveServicesAccountId: cognitiveServices.id
     }
   }
   ```

3. Reference the output `raiPolicies.outputs.raiPolicyName` in the model deployment to bind it (as per RAI-001).

## Verification
1. Deploy the updated Bicep to the target resource group:
   ```bash
   az deployment group create \
     --resource-group <RG> \
     --template-file infra/main.bicep
   ```

2. Verify the content filter resource is live:
   ```bash
   az cognitiveservices account rai-policy list \
     --resource-group <RG> \
     --account-name <foundry-account-name>
   ```

3. Re-run threadlight: `python3 skills/threadlight-production-ready/scripts/production_ready.py --target-rg <RG> --target-sub <SUB>`. RAI-101 should flip from `fail` to `pass` once the `Microsoft.CognitiveServices/accounts/raiPolicies` resource is detected live in Azure.
