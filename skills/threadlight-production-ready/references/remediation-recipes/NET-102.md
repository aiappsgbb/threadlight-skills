## Target file
`infra/foundry/private-endpoint.bicep` (canonical; search for `Microsoft.Network/privateEndpoints` if the path differs).

## Edit type
`replace` — this check verifies that PE resources exist in the target RG and are in an approved state

## Edit recipe

1. This finding depends on **NET-002** (Bicep-side approval state). Ensure NET-002 is addressed first: the `Microsoft.Network/privateEndpoints` resource must declare `privateLinkServiceConnections[0].privateLinkServiceConnectionState.status: 'Approved'` in Bicep.

2. If the PE resource does not exist, create `infra/foundry/private-endpoint.bicep` with a resource block similar to:

```bicep
resource privateEndpoint 'Microsoft.Network/privateEndpoints@2024-01-01' = {
  name: peEndpointName
  location: location
  properties: {
    subnet: {
      id: '${vnet.id}/subnets/${subnetName}'
    }
    privateLinkServiceConnections: [
      {
        name: 'foundry-connection'
        properties: {
          privateLinkServiceId: foundryAccountId
          groupIds: ['account']
          privateLinkServiceConnectionState: {
            status: 'Approved'
            description: 'Auto-approved by threadlight-production-ready v0.4.0 onboarding'
          }
        }
      }
    ]
  }
}
```

3. Ensure the module is referenced from `infra/main.bicep` using a module block (similar to the network module in NET-001).
4. Redeploy the infrastructure to create or update the PE resources:

```bash
azd deploy
```

## Verification

After deployment, re-run threadlight to verify the PE resources exist and are approved:

```bash
python3 scripts/production_ready.py --target-rg <RG> --target-sub <SUB>
```

NET-102 should flip from `must-fix` to `pass` once private endpoints exist in the target RG. The live check queries `az network private-endpoint list -g <RG>` and confirms at least one PE resource is present.

For data-plane approval (if the Bicep change alone doesn't trigger state sync):

```bash
az network private-endpoint-connection approve \
  --id $(az network private-endpoint-connection list \
    --resource-group <RG> \
    --query '[0].id' -o tsv)
```

This command is idempotent and can be run post-deployment as part of the CI/CD pipeline if manual approval is required by your Azure policy.
