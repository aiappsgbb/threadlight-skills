## What this checks
AGT-002 passes when the repo contains a `policy.yaml` (or `agt-policy.yaml` /
`policy.yml`). This is the presence check — AGT-001 then checks that it is
schema-valid. Governance ships as a committed artefact versioned with the code.

## Target file
`policy.yaml` at the repo root (or referenced from `agent.yaml`).

## Edit type
`insert` — create the file if missing.

## Edit recipe
1. At the repo root, create `policy.yaml` with the three required top-level keys
   and one real least-privilege rule:

   ```yaml
   version: "1.0.0"
   name: my-pilot-governance
   description: Least-privilege, default-deny action governance.
   deny_by_default: true
   rules:
     - name: allow-read-only
       priority: 10
       conditions:
         - field: tool.category
           operator: eq
           value: read
       action: allow
   ```
2. If the pilot keeps the policy at a non-standard path, reference it from
   `agent.yaml` and keep it discoverable at the repo root.

## Verification
```bash
ls -la policy.yaml
agt lint-policy policy.yaml     # authors AGT-001 at the same time
```
Re-run threadlight; AGT-002 flips to `pass` once the file is present.
