## Target file
`policy.yaml` (must be at repo root or referenced from `agent.yaml` / `container.py`).

## Edit type
`insert` — create the file if missing.

## Edit recipe
1. At the repo root, create a new file named `policy.yaml`.
2. Populate it with a minimal policy skeleton:

   ```yaml
   # AGT Policy — v0.4.0 Production-Ready Skill Onboarding
   # Reference: https://github.com/microsoft/agent-governance-toolkit/blob/main/docs/policy-schema.md

   version: "1.0"
   agent_control_specification_version: "2026-02-01"
   
   # Tool call governance — restrict mutation operations
   tool_calls:
     allowed_prefixes:
       - "query_"
       - "list_"
       - "read_"
     blocked_patterns:
       - "delete_"
       - "destroy_"
       - "drop_"
   
   # Prompt shield — detect and block jailbreak attempts
   prompt_shields:
     enabled: true
     rules:
       - name: "jailbreak-detection"
         pattern: "ignore previous|system prompt|ignore the policy"
         action: "deny"
       - name: "pii-redaction"
         pattern: "ssn|credit.card|password"
         action: "redact"
   
   # Telemetry
   telemetry:
     sink: "appinsights"
     audit_fields:
       - "tool_name"
       - "decision"
       - "timestamp"
   ```

3. If the repository already has a non-standard policy location (e.g. `agt-policy.yaml` or referenced from `agent.yaml`), use that path instead and ensure it is discoverable at repo root or documented in the SPEC.
