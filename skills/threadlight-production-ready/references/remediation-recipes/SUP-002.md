## Target file
`infra/main.bicep` (or any Bicep file with `module` declarations using `br:` references)

## Edit type
`replace`

## Edit recipe
1. Search all Bicep files for AVM module references:
   ```bash
   grep -r "br:[^']*:latest" --include="*.bicep" infra/
   ```

2. For each module reference using `:latest`, replace with a pinned version tag:
   - **Before:** `br/public:avm/res/key-vault/vault:latest`
   - **After:** `br/public:avm/res/key-vault/vault:0.4.0`

3. To find available versions for an AVM module:
   ```bash
   az bicep list-versions --registry "br/public" --artifact "<module-path>"
   ```
   Example:
   ```bash
   az bicep list-versions --registry "br/public" --artifact "avm/res/key-vault/vault"
   ```

4. Example Bicep module declaration before and after:

**Before:**
```bicep
module keyVault 'br/public:avm/res/key-vault/vault:latest' = {
  name: 'keyVaultDeployment'
  params: {
    name: kvName
    location: location
  }
}
```

**After:**
```bicep
module keyVault 'br/public:avm/res/key-vault/vault:0.4.0' = {
  name: 'keyVaultDeployment'
  params: {
    name: kvName
    location: location
  }
}
```

5. Update **all** module references in your Bicep files. Common AVM modules:
   - `avm/res/key-vault/vault`
   - `avm/res/storage-account`
   - `avm/res/container-apps/container-app-environment`
   - `avm/res/container-registry`

6. Pin to a recent stable version (e.g., `0.4.0` or `0.5.0`), not a pre-release (`0.0.1-alpha`).
