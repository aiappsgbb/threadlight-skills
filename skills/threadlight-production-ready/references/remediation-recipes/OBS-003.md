## Target file
`src/app.py` (or the workload entry point; search for `configure_azure_monitor()` or `from opentelemetry` if path differs). If running Java or .NET, adapt to equivalent init (`ApplicationInsights.Agent` or `OpenTelemetry.Exporter.AzureMonitor`).

## Edit type
`insert`

## Edit recipe
1. Open the workload entry point (`src/app.py` for Python, `Program.cs` for .NET, `main.java` for Java).
2. At the **top** of the file (before importing the framework), insert the Azure Monitor instrumentation:

**Python (before FastAPI/Flask init):**
```python
from azure.monitor.opentelemetry import configure_azure_monitor

configure_azure_monitor()
```

**Java (.NET/Kafka imports):**
```java
import com.azure.monitor.opentelemetry.exporter.AzureMonitorExporterBuilder;
// Configure before creating agent/workload
```

**C# / .NET:**
```csharp
using Azure.Monitor.OpenTelemetry.AspNetCore;

var builder = WebApplication.CreateBuilder(args);
builder.Services.AddOpenTelemetry().UseAzureMonitor();
```

3. Ensure the call happens **before** the framework server starts (before `app.run()` in Python, before `app.Listen()` in .NET, etc.).
4. Test that the app starts without errors: `python src/app.py` or equivalent. Traces should appear in App Insights within 30 seconds.

## Verification
Re-run threadlight: `python3 scripts/production_ready.py --target-rg <RG> --target-sub <SUB>`. OBS-003 should flip from `fail` to `pass`.

After deployment and a few minutes of traffic, check for ingested spans in App Insights:
```bash
az monitor app-insights query \
  --app <app-insights-name> \
  --resource-group <RG> \
  --analytics-query "traces | where timestamp > ago(30m) | take 1"
```

Expected: ≥ 1 row.
