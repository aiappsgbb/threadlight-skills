## Target file
`infra/identity.bicep` (or the `*.bicep` / compiled `*.json` that declares the
agent's identity), plus any source file the scan flagged as carrying a standing
secret — `ClientSecretCredential(...)`, `passwordCredentials`, or
`az ad app credential ... --password`.

## Edit type
`replace`

## Edit recipe
A non-human identity (NHI) backed by a client secret is a standing credential you
have to store, rotate, and can leak. Bind the agent to a **passwordless** identity
instead — a user-assigned managed identity (UAMI) for in-Azure workloads, or a
**federated identity credential** for GitHub Actions / external issuers so no
secret ever exists.

1. **Declare a user-assigned managed identity** and remove the app-registration
   secret block:

   ```bicep
   // BEFORE (BAD) — Microsoft.Graph/applications with a standing password:
   resource app 'Microsoft.Graph/applications@1.0' = {
     displayName: 'agent'
     passwordCredentials: [ { displayName: 'sp-secret' } ]  // REMOVE
   }

   // AFTER (GOOD) — passwordless UAMI:
   resource agentId 'Microsoft.ManagedIdentity/userAssignedIdentities@2023-01-31' = {
     name: 'id-${agentName}'
     location: location
     tags: { owner: ownerEmail }
   }
   ```

2. **For CI / external issuers, use a federated credential** — the workload gets a
   short-lived token from its own issuer, so there is no secret to store:

   ```bicep
   resource fic 'Microsoft.ManagedIdentity/userAssignedIdentities/federatedIdentityCredentials@2023-01-31' = {
     parent: agentId
     name: 'gh-oidc'
     properties: {
       issuer: 'https://token.actions.githubusercontent.com'
       subject: 'repo:${org}/${repo}:ref:refs/heads/main'
       audiences: [ 'api://AzureADTokenExchange' ]
     }
   }
   ```

3. **In code**, drop `ClientSecretCredential` for `DefaultAzureCredential` /
   `ManagedIdentityCredential` (reference IAM-001 for the source-side edit).

4. If you provision the agent's identity through **entra-agent-id** or
   **foundry-agt**, declare it there in passwordless mode — the platform issues
   and manages the identity so no secret is minted.

## Verification
Re-run the producer, then the full readiness check:

```bash
python3 scripts/agent_identity.py --root . --out agent-identity.json
python3 scripts/production_ready.py --target-rg ${TARGET_RG} --target-sub ${TARGET_SUB}
```

IAM-006 passes when every discovered agent identity is `uami` or `federated` and
no standing-secret signal (`ClientSecretCredential`, `passwordCredentials`,
`--password`) remains.
