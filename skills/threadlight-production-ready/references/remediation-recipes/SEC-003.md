## Target file
`appsettings.json`, `appsettings.Production.json`, or `local.settings.json` (search for any JSON config file that references secrets or sensitive connection strings; priority order: `src/appsettings.json` > `src/local.settings.json` > root-level config files).

## Edit type
`replace` — update config sections that contain raw values, replacing them with Key Vault reference syntax.

## Edit recipe
1. Open the config file (e.g., `src/appsettings.json`).
2. Locate sections with connection strings, API keys, or passwords. Examples:

   ```json
   {
     "ConnectionStrings": {
       "DefaultConnection": "Server=myserver;Database=mydb;Uid=user;Pwd=raw_password_here;"
     },
     "ApiSettings": {
       "ApiKey": "sk-1234567890abcdef"
     }
   }
   ```

3. Replace each raw value with a Key Vault reference using the `@Microsoft.KeyVault()` syntax:

   ```json
   {
     "ConnectionStrings": {
       "DefaultConnection": "@Microsoft.KeyVault(SecretUri=https://mykvault.vault.azure.net/secrets/db-connection-string/)"
     },
     "ApiSettings": {
       "ApiKey": "@Microsoft.KeyVault(SecretUri=https://mykvault.vault.azure.net/secrets/api-key/)"
     }
   }
   ```

4. Ensure the secrets exist in the target Key Vault with matching names (`db-connection-string`, `api-key`, etc.). If they don't exist, create them first:

   ```bash
   az keyvault secret set --vault-name <kvname> --name db-connection-string --value "<connection-string>"
   az keyvault secret set --vault-name <kvname> --name api-key --value "<api-key>"
   ```

5. For .NET Azure Functions (`local.settings.json`), use the same syntax:

   ```json
   {
     "Values": {
       "StorageConnectionString": "@Microsoft.KeyVault(SecretUri=https://mykvault.vault.azure.net/secrets/storage-connection/)",
       "CosmosDbConnectionString": "@Microsoft.KeyVault(SecretUri=https://mykvault.vault.azure.net/secrets/cosmosdb-connection/)"
     }
   }
   ```

6. Ensure the application runtime supports Key Vault references (Azure App Service, Azure Functions, Azure Container Apps with managed identity binding). Verify the app has:
   - Managed identity enabled on the compute resource
   - Role assignment `Key Vault Secrets User` on the target Key Vault

7. Save and commit the updated config file.

## Verification
1. Deploy the updated config to the target environment (App Service, Function App, etc.).
2. Verify the application can load config:
   - Check application logs for any `KeyVaultReference` resolution errors.
   - Call an app endpoint that uses the config (e.g., database query, API call).
3. Re-run threadlight: `python3 scripts/production_ready.py --target-rg <RG> --target-sub <SUB>`. SEC-003 should flip from `should-fix` to `pass` (Key Vault references detected in config files).

If the app still cannot resolve references:
- Confirm managed identity is enabled: `az resource show -g <RG> --name <app-name> --resource-type Microsoft.Web/sites --query identity.principalId`
- Confirm role assignment exists: `az role assignment list --scope /subscriptions/<sub>/resourceGroups/<RG>/providers/Microsoft.KeyVault/vaults/<kvname> --query "[?principalId=='<identity-id>']"`
