## Target file
N/A — this fix is a **live operator action** in Azure. There is no single repo-local file to edit. MDL-110 is a pre-flight quota check that runs at deploy time, not a code remediation.

## Edit type
`manual` — The operator must verify or request quota increase via Azure portal or support.

## Edit recipe
1. **Understand the check**: MDL-110 compares your planned TPM (declared in SPEC § 12) against your current Azure quota for the Foundry account in the target region. If `planned-TPM / quota-TPM > --quota-utilization` (default 80%), the check fails.

2. **Retrieve current quota**:

   ```bash
   az cognitiveservices usage list \
     --resource-group <RG> \
     --name <foundry-account-name> \
     --query '[?name.value==`TPM`].[currentValue, limitValue]' \
     -o table
   ```

   Example output: `currentValue: 100, limitValue: 200` (200 TPM quota available, 100 currently used).

3. **Calculate required quota**: From SPEC § 12, identify your planned TPM load. Add 20% headroom to the 80% default threshold.
   - Example: If you plan for 150 TPM, you need at least `150 / 0.8 = 187.5 TPM` quota.

4. **If quota is insufficient**:
   - In Azure Portal: Navigate to **Cognitive Services** → **<Foundry account>** → **Quotas** → **Manage quotas**.
   - Request an increase to the required TPM (e.g., 200 TPM).
   - Or use Azure support: File a support ticket requesting TPM limit increase in `<region>`.
   - Allow 1–5 business days for approval.

5. **If quota is sufficient**: No action needed; MDL-110 should pass on next run.

## Verification
1. After quota increase approval, re-run threadlight: `python3 scripts/production_ready.py --target-rg <RG> --target-sub <SUB> --tier 1 --quota-utilization 80` (requires Cognitive Services Usages Reader role).
2. MDL-110 should flip from `fail` to `pass` (planned TPM ≤ 80% of available quota).
3. Verify quota was increased:

   ```bash
   az cognitiveservices usage list \
     --resource-group <RG> \
     --name <foundry-account-name>
   ```

   Expected: `limitValue` now >= required quota.
