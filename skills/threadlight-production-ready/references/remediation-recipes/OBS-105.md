## Target file

## Edit type
`insert`

## Edit recipe
1. Create or open `infra/observability/action-group.bicep`.
2. If the file is empty or does not exist, insert the action group resource with notification channels:

```bicep
param actionGroupName string
param location string = 'global'
param emailReceiver string = ''  // On-call email from framing
param tags object = {}

resource actionGroup 'Microsoft.Insights/actionGroups@2023-01-01' = {
  name: actionGroupName
  location: location
  properties: {
    groupShortName: 'ThreadLight'
    enabled: true
    emailReceivers: [
      {
        name: 'EmailNotification'
        emailAddress: emailReceiver
        useCommonAlertSchema: true
      }
    ]
    smsReceivers: []
    webhookReceivers: []
    itsmReceivers: []
    azureAppPushReceivers: []
    automationRunbookReceivers: []
    voiceReceivers: []
    logicAppReceivers: []
    azureFunctionReceivers: []
    armRoleReceivers: []
  }
  tags: tags
}

output actionGroupResourceId string = actionGroup.id
output actionGroupName string = actionGroup.name
```

3. Pass the `emailReceiver` parameter from framing (on-call email). If SMS or webhook notifications are required, add `smsReceivers` or `webhookReceivers` blocks with appropriate credentials.
4. Ensure the action group is referenced in the alert rules (OBS-104) and that the resource ID is passed as `actionGroupResourceId`.
