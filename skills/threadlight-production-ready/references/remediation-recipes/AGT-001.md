## What this checks
AGT-001 passes when `policy.yaml` is a **schema-valid** Agent Governance Toolkit
policy — it has top-level `version`, `name`, and `rules:` and therefore lints
clean under `agt lint-policy`. Governance is authored as a committed policy, not
imported as in-process middleware.

## Target file
`policy.yaml` (or `agt-policy.yaml` / `policy.yml`) at the repo root.

## Edit type
`replace`

## Edit recipe
1. Install the toolkit locally (author-time only — this is not a repo dependency):
   ```bash
   pip install agent-governance-toolkit
   ```
2. Give the policy the three required top-level keys and at least one rule.
   Each rule is `{name, priority, conditions:[{field, operator, value}], action}`;
   valid actions are `allow | deny | audit | block | escalate | rate_limit`.
   `deny_by_default: true` makes any unmatched action denied.

   ```yaml
   version: "1.0.0"                 # semver of THIS policy document, never 'latest'
   name: my-pilot-governance
   description: Least-privilege, default-deny action governance.
   metadata:
     references:
       - owasp-asi-2026             # OWASP Agentic Security Initiative reference
   deny_by_default: true
   rules:
     - name: block-shell-exec
       priority: 100
       conditions:
         - field: tool.name
           operator: in
           value: ["shell_exec", "exec", "subprocess"]
       action: block
     - name: escalate-external-api
       priority: 80
       conditions:
         - field: tool.name
           operator: eq
           value: external_api_call
       action: escalate
     - name: allow-read-only
       priority: 10
       conditions:
         - field: tool.category
           operator: eq
           value: read
       action: allow
   ```
3. Fill the tool families from the pilot's `agent.yaml` tool set — the point is a
   real least-privilege ruleset, not a placeholder.

## Verification
```bash
agt lint-policy policy.yaml     # must exit 0 with no errors
```
Then re-run threadlight (`python3 scripts/production_ready.py --root <repo> --static`).
AGT-001 flips to `pass` once the policy has schema-valid `version` + `name` + `rules`.
