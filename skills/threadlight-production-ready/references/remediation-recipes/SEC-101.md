## Target file
`infra/foundry/kv.bicep` (or wherever the `Microsoft.KeyVault/vaults` resource is declared in Bicep — search the repo for `Microsoft.KeyVault/vaults` if the canonical path differs).

## Edit type
`replace` — update the existing Key Vault resource properties to enable soft-delete.

## Edit recipe
1. Open `infra/foundry/kv.bicep` in your editor.
2. Locate the `Microsoft.KeyVault/vaults` resource declaration.
3. Inside the `properties` block, ensure `enableSoftDelete: true` is set:

   ```bicep
   resource keyVault 'Microsoft.KeyVault/vaults@2023-07-01' = {
     name: kvName
     location: location
     properties: {
       tenantId: tenantId
       sku: {
         family: 'A'
         name: 'standard'
       }
       enableRbacAuthorization: true
       enableSoftDelete: true           // <-- Add or replace this line
       enablePurgeProtection: true
       softDeleteRetentionInDays: 90    // <-- Ensure >= 90
       publicNetworkAccess: 'Disabled'
       networkAcls: {
         bypass: 'AzureServices'
         defaultAction: 'Deny'
       }
     }
   }
   ```

4. If `softDeleteRetentionInDays` is missing or less than 90, set it to 90 (production minimum).
5. Save the file and commit:
   ```bash
   git add infra/foundry/kv.bicep
   git commit -m "Enable soft-delete on Key Vault (SEC-101)
   
   - Set enableSoftDelete: true
   - Set softDeleteRetentionInDays: 90
   
   Fixes SEC-101"
   ```

6. Deploy the Bicep changes to Azure:
   ```bash
   az deployment group create \
     --resource-group <RG> \
     --template-file infra/main.bicep \
     --parameters location=<region>
   ```

## Verification
1. After deployment, verify soft-delete is enabled on the live KV:
   ```bash
   az keyvault show -n <kvname> -g <RG> --query "properties.enableSoftDelete"
   ```
   Expected output: `true`

2. Confirm retention days:
   ```bash
   az keyvault show -n <kvname> -g <RG> --query "properties.softDeleteRetentionInDays"
   ```
   Expected output: `90` (or higher)

3. Re-run threadlight: `python3 scripts/production_ready.py --target-rg <RG> --target-sub <SUB>`. SEC-101 should flip from `must-fix` to `pass`.

If the deployment fails or soft-delete remains disabled, check:
- Bicep syntax: `az bicep build --file infra/main.bicep`
- Deployment logs: `az deployment group show -g <RG> -n <deployment-name> --query "properties.outputs"`
- Key Vault API version support (ensure `@2023-07-01` or later supports the `enableSoftDelete` property).
