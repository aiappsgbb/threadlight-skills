## Target file
`infra/observability/diagnostic-settings.bicep` (create if missing; search for `Microsoft.Insights/diagnosticSettings` if path differs).

## Edit type
`insert`

## Edit recipe
1. Create or open `infra/observability/diagnostic-settings.bicep`.
2. If the file is empty or does not exist, insert the diagnostic settings resource:

```bicep
param foundryAccountResourceId string
param logAnalyticsWorkspaceId string
param diagnosticSettingsName string = 'foundry-to-loganalytics'
param tags object = {}

resource diagnosticSettings 'Microsoft.Insights/diagnosticSettings@2021-05-01-preview' = {
  name: diagnosticSettingsName
  scope: resourceGroup()
  properties: {
    workspaceId: logAnalyticsWorkspaceId
    logs: [
      {
        category: 'Audit'
        enabled: true
        retentionPolicy: {
          enabled: true
          days: 90
        }
      }
    ]
    metrics: [
      {
        category: 'AllMetrics'
        enabled: true
        retentionPolicy: {
          enabled: true
          days: 90
        }
      }
    ]
  }
}

output diagnosticSettingsId string = diagnosticSettings.id
```

**Alternative (for Foundry-specific diagnostics):**
If you need to attach diagnostics directly to the Foundry account resource, use a scoped diagnostic settings:

```bicep
param foundryAccountResourceId string
param logAnalyticsWorkspaceId string

resource foundryAccount 'Microsoft.CognitiveServices/accounts@2023-05-01' existing = {
  name: last(split(foundryAccountResourceId, '/'))
}

resource foundryDiagnostics 'Microsoft.Insights/diagnosticSettings@2021-05-01-preview' = {
  name: 'foundry-diagnostics'
  scope: foundryAccount
  properties: {
    workspaceId: logAnalyticsWorkspaceId
    logs: [
      {
        category: 'APIManagement'
        enabled: true
      }
      {
        category: 'ContentModerationRequests'
        enabled: true
      }
    ]
    metrics: [
      {
        category: 'AllMetrics'
        enabled: true
      }
    ]
  }
}

output diagnosticSettingsId string = foundryDiagnostics.id
```

3. Link this module from the main template, passing `foundryAccountResourceId` and `logAnalyticsWorkspaceId`.
4. Ensure retention is set to ≥ 90 days for production compliance (OBS-106 checks for this).

## Verification
Re-run threadlight: `python3 scripts/production_ready.py --target-rg <RG> --target-sub <SUB>`. OBS-106 should flip from `fail` to `pass`.

Confirm diagnostic settings are active:
```bash
az monitor diagnostic-settings list \
  --resource $(az cognitiveservices account list \
    --resource-group <RG> \
    --query '[0].id' -o tsv)
```

Expected: At least one diagnostic setting with `workspaceId` pointing to the Log Analytics workspace and `logs[].retentionInDays` ≥ 90.

Verify logs are arriving in Log Analytics:
```bash
az monitor log-analytics query \
  --workspace <workspace-id> \
  --analytics-query "AzureDiagnostics | where timestamp > ago(1h) | take 1"
```

Expected: ≥ 1 row (if Foundry account has been accessed recently).
