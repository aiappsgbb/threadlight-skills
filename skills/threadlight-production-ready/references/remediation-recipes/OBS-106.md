## Edit recipe
Present this to the operator before proceeding:

> **OBS-106 — Manual diagnostic-settings configuration (sibling skill pending)**
>
> The `azure-resource-diagnostics` skill (tracked at `aiappsgbb/awesome-gbb#271`) will eventually validate + provision diagnostic settings automatically. Until that skill ships, drive it manually:
>
> 1. **Audit current diagnostic settings on the Foundry account:**
>    ```bash
>    FOUNDRY_ID=$(az cognitiveservices account list --resource-group <RG> --query '[0].id' -o tsv)
>    az monitor diagnostic-settings list --resource $FOUNDRY_ID
>    ```
> 2. **If none exist, create one** routing `Audit` + `AllMetrics` to the central Log Analytics workspace with ≥ 90-day retention. Codify it in `infra/observability/diagnostic-settings.bicep`:
>    ```bicep
>    param foundryAccountResourceId string
>    param logAnalyticsWorkspaceId string
>
>    resource foundryAccount 'Microsoft.CognitiveServices/accounts@2023-05-01' existing = {
>      name: last(split(foundryAccountResourceId, '/'))
>    }
>
>    resource foundryDiagnostics 'Microsoft.Insights/diagnosticSettings@2021-05-01-preview' = {
>      name: 'foundry-diagnostics'
>      scope: foundryAccount
>      properties: {
>        workspaceId: logAnalyticsWorkspaceId
>        logs: [
>          { category: 'Audit', enabled: true, retentionPolicy: { enabled: true, days: 90 } }
>        ]
>        metrics: [
>          { category: 'AllMetrics', enabled: true, retentionPolicy: { enabled: true, days: 90 } }
>        ]
>      }
>    }
>    ```
> 3. **Deploy and verify** that logs are arriving:
>    ```bash
>    az monitor log-analytics query \
>      --workspace <workspace-id> \
>      --analytics-query "AzureDiagnostics | where TimeGenerated > ago(1h) | take 1"
>    ```
