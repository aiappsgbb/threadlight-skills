## What this checks
AGT-005 passes when a CI workflow under `.github/workflows/` runs the Agent
Governance Toolkit — `agt lint-policy`, `agt test`, or `agt verify` — so a broken
or weakened policy fails the pipeline instead of reaching production. The gate
lives in CI, not in the agent's request path.

## Target file
`.github/workflows/governance.yml` (new), or an existing pipeline job.

## Edit type
`insert` — add a new workflow (or a job to an existing pipeline).

## Edit recipe
1. Add a job that installs the toolkit and lints + tests the committed policy:

   ```yaml
   name: agent-governance
   on: [pull_request, push]
   jobs:
     governance:
       runs-on: ubuntu-latest
       steps:
         - uses: actions/checkout@v4
         - uses: actions/setup-python@v5
           with:
             python-version: "3.12"
         - run: pip install agent-governance-toolkit
         - run: agt lint-policy policy.yaml
         - run: agt test policy.yaml        # runs policy unit tests if present
   ```
2. To emit a signed attestation on the protected branch, add
   `agt verify --badge --evidence governance-evidence.json` and commit the
   resulting `governance-attestation/v1` artefact.
3. If you already deploy through `threadlight-cicd`, add the governance step to
   that workflow rather than creating a second pipeline.

## Verification
```bash
grep -rEl 'agt +(verify|lint-policy|test)|agent-governance-toolkit' .github/workflows/
```
Re-run threadlight; AGT-005 flips to `pass` once a workflow invokes the toolkit.
