## Target file
`infra/identity.bicep` (or the resource that declares the agent identity), or the
optional `agent-identity.governance.json` manifest at the repo root.

## Edit type
`replace`

## Edit recipe
A standing identity with no review date lives forever by default — long after the
agent it served is gone. Give every identity a lifecycle: a review/expiry date, or
better, remove the standing credential entirely by federating.

1. **Best — federate so there is no standing secret to review.** A federated
   identity credential mints short-lived tokens on demand; it satisfies IAM-009
   with no expiry tag needed (see IAM-006 for the federation block).

2. **Otherwise, declare a review or expiry date** on the identity (any of
   `reviewBy`, `expiresOn`, or a `reviewBy` entry in the governance manifest):

   ```bicep
   resource agentId 'Microsoft.ManagedIdentity/userAssignedIdentities@2023-01-31' = {
     name: 'id-${agentName}'
     location: location
     tags: {
       owner: 'ai-platform@contoso.com'
       reviewBy: '2026-12-31'   // ADD — when this identity must be re-attested
     }
   }
   ```

3. **Enforce the review** with an Entra **access review** / **PIM** on the
   identity's group or role assignments, so the date is acted on rather than
   decorative:

   ```bash
   # Create a recurring access review for the workload identity's group
   az rest --method POST \
     --url 'https://graph.microsoft.com/v1.0/identityGovernance/accessReviews/definitions' \
     --body @access-review.json
   ```

4. If you register the identity through **entra-agent-id**, set its review cadence
   there; this repo inherits it via `agent-identity.governance.json`.

## Verification
Re-run the readiness check:

```bash
python3 scripts/production_ready.py --target-rg ${TARGET_RG} --target-sub ${TARGET_SUB}
```

IAM-009 passes when every standing identity (`uami` / `app-secret`) carries a
`reviewBy` / `expiresOn` signal or a manifest review entry — federated identities
pass automatically because they hold no standing secret.
